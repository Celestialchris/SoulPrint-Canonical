Memory Passport is SoulPrint’s exportability and integrity surface for continuity. It packages canonical records with provenance metadata so exports can be inspected and validated, without replacing canonical local records.

What Memory Passport is

Memory Passport is a provenance-and-validation construct layered over canonical truth. It supports inspectability, continuity, and local ownership by describing what was exported and where each unit originated.

It does not replace the canonical local ledger. Canonical records in SoulPrint remain the source of truth; passport artifacts are export snapshots and integrity evidence.

What export does

Export writes an inspectable package from canonical records. Canonical data is copied into a structured artifact, while derived materials (like markdown views) are included as non-canonical representations tied back to provenance.

Export is currently available through the existing CLI.

python -m src.passport.cli export <output_dir>

What validation does

Validation checks passport package integrity and internal coherence: required files, record structure, lane/provider consistency, and provenance links.

  • Valid: integrity checks passed without diagnostics.
  • Valid with warnings: core integrity passed, with non-blocking issues to review.
  • Invalid: one or more blocking integrity failures were detected.

python -m src.passport.cli validate <passport_path>
