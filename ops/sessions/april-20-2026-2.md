**Decisions:**
- Migration guards are non-optional for new columns. `db.create_all()` does not alter existing tables. Every `db.Column` addition from this point forward must ship with an explicit `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` guard or equivalent, mirroring the pattern the `is_archived` column established. Codex P1 review caught this on PR #124; Cowork's own stop-condition list had flagged it but proceeded anyway. Stop conditions are advisory, not blocking. The review bot is the actual second line of defense.
- Test tempdirs that interact with `Path.home()` validation must use home-relative paths on CI. `make_test_temp_dir` places under `./.tmp-tests/` which lands on D: drive on Windows runners but under /home/runner on macOS/Linux. Tests that validate paths under home pass on macOS/Linux by coincidence and fail on Windows. Fix: construct a Path.home()-relative tempdir for those specific tests, cleanup via `addCleanup(shutil.rmtree, ...)`. Pattern added to test hygiene.

**Learned patterns (promote to ops/learned/ if recurring):**
- Branch divergence is not deletion. If a branch was cut from main at commit X before feature Y landed on main, feature Y's files are absent from that branch's working tree. This looks like files "disappearing" when switching branches. Twice today this triggered recovery panic. The fix is always `git merge main` (or `git rebase main`) onto the feature branch. Not `git reset`. Not cherry-picking. Not "why is Claude Code deleting my files." Every future instance of "files disappeared" starts with `git branch --show-current` and `git log --oneline -5`.
- Squash-merges do not carry commits added to a PR branch after the PR was opened unless the PR explicitly refreshes. Commits pushed post-PR-open sometimes get absorbed, sometimes get silently dropped. For documentation-critical commits (like the context layer), verify the PR "Commits" tab on GitHub shows the commit count you expect before clicking merge. Alternative: open a fresh PR for any commits added late.
- When the drafter (Claude strategy chat) pre-stages infrastructure for a future task (e.g., factoring the scan function for P7 CLI consumption before P7 exists), do it in the same PR as the feature that first needs it. Don't split across PRs unless the infrastructure has independent test value.
