## Decisions
- `.assets/` directory name derives from the final chosen `.md` filename stem (post-disambiguation), not from the raw `assets_stem`. This keeps the directory name in sync with the `.md` file when collisions occur, at the cost of a mismatch with pre-rendered markdown links in the collision case. Acceptable for this branch; marker-link parity in collision scenarios is a later concern.
- Missing physical asset files raise `FileNotFoundError` (caught as `OSError`) and redirect with `export_error`. The `.md` file is left in place — it was already written. A broken bundle is reported, not silently produced.
- Asset bundle writes in the multi-select loop are inside the existing `try: ... except OSError` block, so they participate in the same error-reporting path as `.md` write failures.
- Tests create physical asset files at `app.instance_path / asset.storage_path` and remove them in `addCleanup`. The test directories (`assets/sha256/aa/` etc.) are left under the real instance dir but are gitignored.
- Manifest is deterministic: assets sorted by `(attached_at_unix, id, asset_id)`, matching the renderer's sort order. No absolute machine paths in manifest values.

## Codex review fix

Codex flagged that markdown links could point to a stale `.assets` folder when `_pick_unique_filename(...)` disambiguated the exported markdown filename. Fixed by rendering markdown with the final chosen asset stem and copying files into the matching sibling `.assets` folder.

Concretely: `render_conversation_markdown(...)` now accepts `assets_stem_override=None`. Directory export routes use a two-pass pattern: first render to obtain the base filename, call `_pick_unique_filename()` to get the final name, derive the stem from that name, then re-render with `assets_stem_override=stem` so embedded `[[stem.assets/...]]` links match the actual copied folder. Browser download and zip mode are unchanged.

Added `MultiSelectExportDirectoryBundleCollisionTest` (2 tests) to `test_multi_select_export.py`:
- `test_each_md_links_to_its_own_assets_folder`: proves each `.md` references a `.assets/` folder that exists and contains its copied files.
- `test_no_md_points_to_nonexistent_assets_folder`: regex-scans all `.md` link targets and asserts every referenced `.assets/` folder is present on disk.
