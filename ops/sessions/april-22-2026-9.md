Tests at `tests/test_tags.py` cover normalization, auto-tag extraction, route behavior, sanitizer rejection, and basic rendering.
**Decisions:**
- Row-action CSS class promotion (`.row-action-btn`) NOT in this PR. Chips ship with their own `.tag-chip*` classes because tag chips are genuinely more complex UI than single-color star/archive buttons, not because star/archive got promoted. Star/archive remain inline-styled per the CP1 pattern; promotion happens in a separate mini-PR after A1 merges.
- Auto-tag only at the persistence-layer construction site, not per-adapter. All five provider importers funnel through `persist_normalized_conversations` at persistence.py:67, so one wire-in point covers all.
- `normalize_tag_string` called twice in the add route (once on incoming raw tag, once on combined string) to dedupe against existing tags. Cheap and explicit; avoids hand-rolling dedupe logic in the route.
- No retroactive auto-tag pass on already-imported rows. Users who want tags on existing conversations add them inline; spec v2 Section 4 explicitly accepts this.
- Import in `persistence.py` uses absolute form (`from src.app.tags import auto_tag_from_title`) per the python-patterns.md cross-package rule. Route-level imports in `__init__.py` use same-package relative form (`from .tags import normalize_tag_string`).

If tags feel sufficient as-is, next work is row-action CSS class promotion (small refactor PR) or Phase B star-as-triage (if starring still feels destination-less).
