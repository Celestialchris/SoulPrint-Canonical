## April 22, 2026 — Workspace right-panel fixes: zero-count providers + slug-to-CSS-var conversion
**Branch:** fix/workspace-provider-panel
**What:** Fixed two bugs in Workspace right panel. (1) Archive Status panel now renders all five providers including zero-count ones (Grok was invisible for users with no Grok imports). Added a tail loop in `workspace.py` that appends 0-count entries for any `PROVIDER_DISPLAY_NAMES` slug not present in the GROUP BY result. Added `providers_connected_count` field to `WorkspaceSummary` to keep the "N providers connected" summary accurate. (2) Claude Code's lane dot rendered invisibly because the template interpolated `var(--lane-claude_code)` (underscore) while the CSS variable is `--lane-claude-code` (hyphen). Fixed via `|replace('_', '-')` at both interpolation sites.
**Decisions:**
- Used inline `|replace` filter rather than introducing a `provider_css_slug` Jinja filter or helper. The slug-to-CSS-var transformation appears in two places currently; if it expands to three or more, promote to a filter. This follows the small-reversible-change pattern.
- Zero-count entries appended to the tail of `provider_recent`, not interleaved or front-loaded. Active providers keep their count-descending order at the top; inactive ones cluster at the bottom in a stable, predictable order. Matches what users expect from "sorted by usage, everything else at the end."
- `PROVIDER_DISPLAY_NAMES` used for the extension source rather than `SUPPORTED_IMPORT_PROVIDERS` because the former's dict iteration gives a stable order, while the latter is a frozenset with no guaranteed iteration order.
- Scope lock didn't include `tests/test_workspace_viewmodel.py` — two pre-existing assertions on `provider_recent` conflicted with the new semantics. Drafter audited `test_workspace_home.py` but missed the viewmodel-specific test file. Targeted assertion updates + one rename (`test_provider_recent_empty_when_no_imports` → `test_provider_recent_all_zero_count_when_no_imports`). No new test infrastructure needed.
