## Decisions
- Single-conv browser export remains plain `.md` when no attachments exist. The zip branch is taken only when the conversation has at least one conversation-level or message-level asset.
- The zip filename for single export uses the same stem as the markdown (e.g. `Bundle Download Test.zip`).
- Multi-select browser zip re-renders with `assets_stem_override` only for conversations that have assets. Conversations without assets use the pre-rendered content to avoid the redundant render call.
- `_write_asset_bundle_to_zip` is a peer to `_write_asset_bundle`, not a wrapper. They share the same sort key, manifest schema, and filename conventions. Keeping them separate avoids abstracting over two different output media (Path vs ZipFile).
- `_get_markdown` test helper absorbs the text/markdown vs zip duality so all marker-content assertions remain unchanged.

## Codex review fix (P2)

Codex flagged that `_write_asset_bundle_to_zip` calls `asset_absolute_path`, which raises `ValueError` (not `OSError`) when `storage_path` escapes the instance root via path traversal. The original `except OSError` handlers in both browser routes would let this propagate to a 500.

Fixed by widening both browser-route `except` clauses to `except (OSError, ValueError)`. Added two new tests (one per route) that mock `asset_absolute_path` to raise `ValueError` and confirm the redirect-with-error behavior.
