# Session: April 24, 2026 — PR next-2 (ImportRun table + instrumentation)

## What shipped

**Observable Archive v0, 2/4.** Every import attempt now creates a durable `ImportRun` row in SQLite.

### New model: `ImportRun` (src/app/models/__init__.py)

Ten columns: id, provider, filename, imported_at (unix float, indexed), status (indexed), conversations_imported, messages_imported, conversations_skipped, conversations_failed, error_message. Created via `db.create_all()` on next app start — no ALTER, no migration.

### New service module: `src/app/import_runs.py`

Three public functions:

- `classify_import_outcome(imported, skipped, failed, reached_importer) -> str` — pure function, no DB. Rules in evaluation order: not reached → failed; failed+others → partial; failed only → failed; imported > 0 → success; skipped only → duplicate_only; all-zero → failed.
- `record_import_run(*, ...)` — inserts and commits one row. Truncates error_message to 500 chars.
- `latest_import_runs(limit=10)` — newest-first query for template rendering.

### Two route wire-ins (src/app/__init__.py)

**`/import` POST**: outer try/finally wraps the entire POST block. Accumulators initialized before `try:`, set as results come in, then `classify + record` fires in the finally regardless of return path (redirect, fall-through, or re-raise). Inner try/finally (temp file cleanup) untouched.

**`/imported/scan-claude-code` POST**: same outer try/finally pattern. All three early-redirect paths (no sessions, bad path, no sessions match) set `run_error_message` before redirecting; the finally always fires. `run_filename` starts as `"scan-claude-code"`, updated to `"scan-claude-code:{N}-sessions"` once `to_import` is known.

### Template update: `src/app/templates/import.html`

"Recent imports" section injected after the outer `{% endif %}` on the three-branch block, before `{% endblock %}`. Renders on all three branches (result, error, and the upload form). Empty state: `"No imports yet."`. Table shows: when, provider (via `provider_display_name` filter), file, status, counts. No new CSS classes.
