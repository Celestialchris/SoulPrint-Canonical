## netlify.toml at repo root

New file with the minimum config that locks the deploy contract:

```toml
[build]
  publish = "landing"
```

Three lines exactly: `[build]` table header, two-space indented `publish = "landing"`, blank line at EOF. No build command, no base directory, no functions directory.

**Why:** soulprint.dev is hosted on Netlify (site slug `superb-pothos-0472e9`). Auto-deploy via the GitHub App webhook is currently broken; manual dashboard retries also fail with an empty deploy log. The empty log suggests Netlify is reading ambiguous dashboard state. Versioning a minimal `netlify.toml` at the repo root locks the publish directory explicitly, eliminating the "Netlify cannot decide what to deploy" failure mode and surviving any future GitHub App re-link. A verbose form with empty `command`, `base`, and `functions` keys was considered and rejected because empty-string values can confuse Netlify; the minimal `[build] publish = "landing"` shape is cleanest.
