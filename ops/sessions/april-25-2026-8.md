# Session: April 25 2026 - feat/message-attachment-ui

## Branch
`feat/message-attachment-ui`

## What shipped
Phase 3 of Session Attachments: message-level attachment upload, listing, and download on the imported conversation explorer.

## Decisions
- Download route scopes by both `message_id` and `conversation_id` — the message lookup uses `filter_by(id=message_id, conversation_id=conversation_id)`, so a wrong conversation or a message that belongs to a different conversation returns 404.
- Duplicate same-message upload uses a fast-path pre-check (`_find_message_asset`) before inserting, with an `IntegrityError` catch for the race case. No DB-level unique constraint was added to `MessageAsset` — the pattern is sufficient for MVP.
- Message attachment UI placed inside each message `<article>` block, after `<pre>{{ message.content }}</pre>`, using existing classes (`inline-action`, `ghost-link`, `toc-list`, `metadata-line`). No new CSS added.
- Same bytes uploaded to different messages reuse one `Asset` row and create separate `MessageAsset` rows, consistent with the ledger's content-addressed storage design.
- Markdown export, Obsidian export, parsing, OCR, and embeddings are not touched — those are Phase 4.
