## April 21, 2026 — CP1: Starring conversations + notes

**Branch:** feat/starring-everywhere
**Version:** v0.7.0-alpha.3 (no bump)

**What shipped:**
CP1 landed on feat/starring-everywhere. Added `is_starred` Boolean column to both `ImportedConversation` and `MemoryEntry` (nullable=False, default=False, server_default="0", index=True). Column guard extended with two new ALTER TABLE checks: one for `imported_conversation.is_starred` (reuses the existing `_cols` PRAGMA result), one for `memory_entry.is_starred` (requires a second `_mcols = PRAGMA table_info(memory_entry)` query since the guard previously only covered the imported_conversation table). Four POST routes added after `unarchive_imported_conversation`: `/imported/<id>/star`, `/imported/<id>/unstar`, `/memory/<id>/star`, `/memory/<id>/unstar`. All use `_safe_next()` helper to honor a `next` form field, falling back to `federated_browser` or `chats`. All set `session["export_notice"]`. `FederatedReadResult` dataclass gained `starred: bool = False`; both lanes populate it from `row.is_starred`. Star toggle UI added to five surfaces: federated browse rows, imported list row actions, chats notes list, imported_explorer page header, memory_detail page header. New test file `tests/test_starring.py`: 13 tests across five classes, all passing.

**Decisions:**
- Two-table schema: `MemoryEntry` gets its own `is_starred` (not a generic flag) to stay consistent with the explicit per-model pattern established by `is_archived` on `ImportedConversation`.
- `next`-param deviation from archive: archive routes redirect to hardcoded endpoints; star routes use `_safe_next()` to return users to their filtered view. This is intentional and not a pattern divergence to fix -- archive's hardcoded redirect is a property of archive semantics (conversation leaves the default list), while starring is non-destructive and the user should stay where they were.
- Memory notice via `session.pop` in template: the `/chats` route body is scope-locked (no modifications to existing routes allowed), so `view.html` calls `session.pop("export_notice", none)` directly. This is valid Flask/Jinja2 -- `session` is a template global. Avoids needing a parallel notice key.
- `starred: bool = False` on `FederatedReadResult` as a typed field rather than piggybacking `source_metadata`. Cleaner: the dataclass is the right place for this, `source_metadata` is a stringly-typed legacy surface.
- `imported_list.html` uses a standalone inline `<form>` for the star toggle (not `formaction`), since the star route needs a `next` field scoped to just that button -- `formaction` can't carry a hidden input without polluting the parent bulk-export form.

**Deferred:**
- FTS snippet rows on `/federated` (lines 93-128): no star badge on message-level results. Scope-excluded.
- "Starred only" filter on any page.
- Star in archived list view.
- Tagging / auto-tag on note creation.
