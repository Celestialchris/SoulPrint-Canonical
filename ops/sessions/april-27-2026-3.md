## Goal

Close the deferred P1 caveat tracked in `ROADMAP.md` § Parked and called out in `SECURITY.md` line 60: when `SOULPRINT_LLM_PROVIDER=openai` and `SOULPRINT_LLM_BASE_URL` was set without `SOULPRINT_LLM_API_KEY`, `provider_from_config()` would assign a dummy `"ollama"` key and return a working `OpenAIProvider` regardless of whether the URL was loopback. A misconfigured non-loopback `BASE_URL` could therefore receive prompts under a fake credential.

The fix gates that single keyless fallback path on a loopback host check. Explicit BYOK to any base URL is preserved.

## What changed

### `src/intelligence/provider.py`

Two top-level stdlib imports added: `ipaddress` and `urllib.parse.urlparse`.

New helper `_is_loopback_base_url(url)` placed immediately above `provider_from_config()`. Algorithm:

1. `urlparse(url).hostname` to extract the host (urllib lowercases and strips IPv6 brackets).
2. If the host is `"localhost"`, accept.
3. Otherwise try `ipaddress.ip_address(host).is_loopback`, which covers every IPv4 in `127.0.0.0/8` and IPv6 `::1`.
4. Any failure (malformed URL, unparseable host, non-IP non-localhost name, missing host, exception from `urlparse`) returns `False`.

No DNS resolution. No network calls. The check is purely textual.

`provider_from_config()` now requires the keyless branch to satisfy three conditions before assigning the `"ollama"` dummy key: provider name is `openai`, base URL is non-empty, and the helper accepts the URL. When the helper rejects, the function returns `None` (the same outcome as for any other unsupported keyless configuration).

`OpenAIProvider.__init__`, `provider_name`, `summarize()`, `complete()`, the model-missing warning, and the Anthropic and stub paths are untouched.

### `tests/test_intelligence_provider.py`

New class `KeylessLoopbackGuardTest` with a `_provider_for(base_url, api_key=None)` helper that scopes env vars cleanly. Coverage:

- Keyless accepts: `localhost`, `127.0.0.1`, another `127.x.x.x` address (`127.0.0.5`), bracketed IPv6 `[::1]`.
- Keyless rejects: public host (`https://api.example.com/v1`), LAN IPv4 (`192.168.1.10`), non-loopback IPv6 (`[2001:db8::1]`), malformed URL (`"not a url"`).
- Explicit-BYOK accepts the same non-loopback URL when `SOULPRINT_LLM_API_KEY` is set; same for the LAN IP case.

Two cases added to the existing `IsLlmConfiguredTest`:

- `is_llm_configured()` returns `False` for keyless non-loopback `BASE_URL` (the gate flowing through to the convenience check).
- `is_llm_configured()` returns `True` for keyless loopback `BASE_URL` (the documented Ollama path still works).

Existing tests in the file (including `OllamaFallbackTest`, the env-var reads, the model-missing warning tests, and the Anthropic and OpenAI summarize/complete max-token tests) were not modified.

## Decisions

1. **Guard only the keyless fallback path, not all configurations.** Alternative: reject every non-loopback `BASE_URL`. Rejected because users may legitimately point an OpenAI-compatible cloud gateway (BYOK) at a non-loopback URL; that is BYOK, not silent-key fallback, and breaking it would harm real users instead of closing the leak.
2. **Implement the helper in `provider.py`, not in a new module.** Alternative: put it under `src/intelligence/` as its own file. Rejected because it has exactly one caller, depends only on stdlib, and would otherwise force test imports to traverse a new path. Single-use helper, single home.
3. **Use `ipaddress.is_loopback` instead of string-matching `127.` prefixes.** Alternative: `host.startswith("127.")` plus a `localhost`/`::1` allowlist. Rejected because `is_loopback` covers the entire `127.0.0.0/8` range correctly, handles IPv6 `::1`, and is unambiguous about `127.0.0.1.evil.com` (which is a hostname, not an IP, and therefore correctly rejected).
4. **Top-level imports for `ipaddress` and `urllib.parse`.** Alternative: lazy import inside the helper, matching the lazy `import logging` and SDK imports in this file. Rejected because both are stdlib, both are tiny, and the helper is on the hot config path; the lazy imports in this file exist for heavy SDKs (`anthropic`, `openai`) and a single warning-only `logging` use, not for stdlib utilities.
