## Decisions
- Conversation-level attachment link format: `[[Title.assets/sha256[:12]-filename.ext]]`. SHA256 prefix makes links unique even when two assets share the same original filename.
- Message-level link format: `[[Title.assets/msg-NNN-sha256[:12]-filename.ext]] · mime/type · sha256`. Sequence index zero-padded to 3 digits.
- Attachments are sorted by `(attached_at_unix or 0, id or 0, asset_id or 0)` before rendering — deterministic across runs regardless of ORM return order.
- `assets_stem` (used in links) is derived from the same title sanitization logic as the markdown filename, computed at the top of the function so both are consistent.
- No filesystem access in the renderer — attachment bytes and physical paths are not touched.
- Obsidian export and browser zip behavior are unchanged; only the markdown content may now contain attachment markers.
